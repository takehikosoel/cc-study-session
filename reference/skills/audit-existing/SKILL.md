# 既存資産の確認（Audit Existing）

## ルール

1. **タスクリストを作る前に既存ファイルを確認する** — Glob/Grepで既に存在するものを把握してから計画する
2. **既存コードを読んでから修正を提案する** — 読まずに「こう書き換えましょう」と言わない
3. **ユーザーが既に準備したものを見落とさない** — CSV、設定ファイル、ドキュメント等が既にあるかチェックする
4. **重複するファイルや関数を作らない** — 似た機能が既にないか検索してから新規作成する

## やること

新機能の実装を始める前に：
```
1. Glob で関連ファイルを検索
2. Grep で関連する関数名・変数名を検索
3. 既存の実装パターンを確認
4. 再利用できるものをリストアップ
5. その上で「新しく作るもの」だけをタスク化
```

## よくある間違い（過去の教訓）

- ユーザーが既にCSVを用意しているのに「CSVを作成する」をタスクに入れた
- 既存のユーティリティ関数があるのに同じ機能の新関数を作った
- 設定ファイルが既にあるのに「設定ファイルを作成する」をTODOに入れた
