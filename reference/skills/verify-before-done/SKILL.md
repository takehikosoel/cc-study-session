# 実装後の動作確認（Verify Before Done）

## ルール

1. **コードを書いたら必ず実行する** — 書いただけで「完了」としない
2. **エラーが出たら自分で直す** — ユーザーに聞く前に修正を試みる（最大3回）
3. **変数名・キー名の一致を確認する** — 特に複数ファイルにまたがる場合、大文字/小文字の統一を検証
4. **サブエージェントが書いたコードも検証対象** — 並行実装後にキー名やインターフェースの不一致がないかチェック

## チェック項目

- [ ] 新しい関数を呼び出し元で正しく使っているか
- [ ] 戻り値のプロパティ名（大文字/小文字）が呼び出し元と一致しているか
- [ ] 外部API呼び出しのレスポンスフィールド名が正しいか
- [ ] 日付フォーマット、タイムゾーンの扱いが正しいか
- [ ] nullやundefinedのハンドリングが漏れていないか
