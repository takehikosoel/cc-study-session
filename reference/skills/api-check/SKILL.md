# API連携の事前確認（API Check）

## ルール

1. **APIのフィールド名を推測しない** — 公式ドキュメントまたは既存コードから正確な名前を確認する
2. **モデル名・バージョンを推測しない** — Claude API、Gemini等のモデルIDは公式から取得する
3. **レスポンスの型を確認する** — 文字列/数値/配列の区別、nullが返る可能性を事前に把握する
4. **レート制限を確認する** — リトライ処理とバックオフを実装する

## やること

- 既存コードにAPI呼び出しがあれば、まずそのパターンを踏襲する
- 新しいAPIエンドポイントを使う場合、WebFetchでドキュメントを確認してから実装する
- パラメータ名が不明な場合、ユーザーに「この情報を確認してください」と聞く（推測で書かない）

## よくある間違い（過去の教訓）

- Meta APIの `impressions` と `views` の混同
- Geminiのモデル名を間違える
- Meta APIのアプリ用途（use case）を間違える
- Google Sheets APIのセル範囲指定ミス
- Chatwork APIのレスポンスフィールド名の誤り
