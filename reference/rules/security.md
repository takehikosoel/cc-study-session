# セキュリティルール（全プロジェクト共通）

## 必ず守ること

### 1. 機密情報の扱い
- API Key、トークン、パスワードをコードにハードコードしない
- 環境変数、Script Properties、.envファイルで管理する
- .envファイルをgitにコミットしない（.gitignoreに追加）

### 2. 危険なコマンドの禁止
- `rm -rf` による一括削除を実行しない
- `git push --force` を確認なしに実行しない
- `git reset --hard` を確認なしに実行しない
- `chmod 777` でパーミッションを開放しない
- `curl` / `wget` で外部にデータを送信しない

### 3. ファイルアクセス制限
- `.env`、`.env.*` ファイルの内容を読み取らない・表示しない
- `*.pem`、`*.key` 秘密鍵ファイルにアクセスしない
- `~/.aws/credentials`、`~/.ssh` にアクセスしない

### 4. コード品質
- ユーザー入力のサニタイズを必ず行う
- SQLインジェクション、XSS、コマンドインジェクションを防止する
- 外部APIレスポンスは必ずバリデーションする

### 5. 権限の最小化
- 必要最小限の権限でツールを実行する
- `Always allow` の乱用を避け、都度確認を心がける
- 不要になった権限は定期的に棚卸しする
